    hexo+github博客问题总结

    基于hexo3-hexo主题搭建过程中遇到的问题以及解决思路

    1.如何更改“关于”界面的内容

    本地站点中依次打开source>about>index.md文档进行修改

    2.如何修改原主题自带的评论

    打开本地站点中主题_config.yml配置文件，修改gitalk:

    3.如何修改“打赏”功能里二维码

    主题文件layout>_partial下的donate.ejs（dashang.ejs）修改内容

    4.如何解决每次更新博文要重新绑定域名

    在本地站点（不是主题文件）source目录下存放CNAME文件（无后缀，内容只写域名：xxx.com）每次部署就会在库中自动添加，favicon.ico图片也可以放在这个文件夹下，这样每次重新部署就不会更新CNAME文件

    5.markdown写博文如何添加图片

    添加图片有两种方式，此处只记录用图床外链添加，其它方式点击这里查看，

    第一步：需要有个挂载图片的图床，我是用的是sm.ms

    第二步：把你的要用的图片上传就好

    第三步：复制网站给你的链接添加到你的markdown博文中，markdown文本也支持html格式，你可以通过html格式链接控制图片显示的大小

    6.如何删除已经发布的博文

    在本地站点文件source>_posts中找到想要删除的博文直接删除，然后再重新部署更新博客就好（没有找到专门删除博文的hexo命令）

    7.博文上传报错：提示ERROR Process failed：

    检查博文格式是否正确，比如头部格式标签少空格或者少符号,都是细节问题

    如何在3-hexo主题中更新文章

    1. 在自己创建的blog文件夹中右击鼠标打开hexo命令框

    2. 要创建markdown文本输入命令：

      hexo n “文章名称”

    3. 通过vscod或其它编译器打开编译

    4. 上传文章 在hexo命令框中输入：

      hexo g
      hexo s
      hexo d

    常用的Hexo 命令

    npm install hexo -g #安装Hexo
    npm update hexo -g #升级
    hexo init #初始化博客

    命令简写

    hexo n “我的博客” == hexo new “我的博客” #新建文章
    hexo g == hexo generate #生成
    hexo s == hexo server #启动服务预览
    hexo d == hexo deploy #部署

    hexo server #Hexo会监视文件变动并自动更新，无须重启服务器
    hexo server -s #静态模式
    hexo server -p 5000 #更改端口
    hexo server -i 192.168.1.1 #自定义 IP
    hexo clean #清除缓存，若是网页正常情况下可以忽略这条命令

    刚刚的三个命令依次是新建一篇博客文章、生成网页、在本地预览的操作。

    头部设置

    在每篇利用 Hexo 创建的文章的开头，都会有对文章进行说明的文字，叫做 文章头部

    title: 文章标题
    date: 编译时间
    tags: 标签
    categories: 所属主题（如Java类文章）

    推荐

    Notepad++
    非常好用的文本编译器，小巧又强大，可以打开很多格式的文件

    Visual Studio Code 非常强大的轻量级编译器，支持各种编译语言，强大的插件扩展，用它写markdowm文本也非常不错（Ctrl+K 再按V 调出实时预览）
